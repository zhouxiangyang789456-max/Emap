# Unity 地形地图编辑器 — 操作手册

本手册从零开始，覆盖安装、地形编辑、2D/3D 双模式、单位放置、保存加载全流程。

---

## 第一部分：安装与导入（2 分钟）

### 步骤1：确认 Unity 版本

打开 Unity Hub，确认你的 Unity 版本是 **2020.3 或更高**。

### 步骤2：创建或打开项目

- **新建项目**：Unity Hub → New Project → 选 "3D Core" → Create
- **已有项目**：直接打开

### 步骤3：导入插件文件

把本插件的 `Assets` 文件夹拖入项目根目录，等待 Unity 编译完成。

---

## 第二部分：创建地形配置表（3 分钟）

### 步骤4：创建配置文件

1. Project 窗口 → 右键 → **Create → 地图编辑器 → 地形资源配置表**
2. 改名为 `MapConfig`，选中它

### 步骤5：创建地形——"草地"

1. 在 Inspector 中点击 **"+ 创建新地形"**（表单式创建）
2. 填写：

```
地形 ID（必填）:  grass
显示名称:         草地
通行规则:          Passable
预览图标（可选）:   （可拖入精灵图片）
```

3. 点击 **"确认创建"**。地形出现在下方的折叠列表中。

### 步骤6：添加更多地形

同样方式创建：
- `stone` / 山石 / Impassable（不可通过）
- `shoal` / 浅滩 / UnitImpassable（部分单位不可通过）

### 步骤7：编辑地形详情

点击折叠列表中的地形标题展开，可编辑：
- 自定义属性（攻击加成、移速加成、是否中毒等 Int/Float/Bool 属性）
- 禁止通行单位 ID 列表（仅 UnitImpassable 类型）
- 预览图标更换

---

## 第三部分：打开地图编辑器（1 分钟）

### 步骤8：打开编辑器窗口

菜单栏 → **地图编辑器 → 打开地图编辑器**

### 步骤9：关联配置表

将 `MapConfig` 从 Project 窗口拖入编辑器的"地形资源配置表"字段。

---

## 第四部分：编辑第一张地图（5 分钟）

### 步骤10：设置地图参数

```
地图名称:         MyFirstMap
地图宽度:         30
地图高度:         20
格子尺寸:         1.0（可用下方预设按钮快速切换 16/32/64/100/128）
```

### 步骤11：3D 编辑模式

- Scene 视图显示**绿色网格**，原点有**青色箭头**
- 操作：右键拖拽旋转、滚轮缩放、中键平移
- 调整到**俯视角度**编辑最方便

### 步骤12：画地形

1. 在下拉框选择 `[grass] 草地`
2. 编辑模式选 **笔刷**
3. 在 Scene 视图格子上**左键点击** → 绿色半透明色块出现
4. **按住左键拖拽** → 连续铺设
5. 切换 `[stone] 山石` → 红色色块

### 步骤13：擦除

- 编辑模式切换到 **橡皮**，左键点击擦除
- 或在笔刷模式下**按住 Shift** 临时切换橡皮

### 步骤14：矩形填充 + 洪水填充

- **矩形填充**：选择模式 → 按住左键从一角拖到对角 → 松开填充整个矩形
- **洪水填充**：选择模式 → 点击一片连续区域的中央 → 整片替换

### 步骤15：撤销 / 重做

- **Ctrl+Z** 撤销 / **Ctrl+Y** 重做
- 编辑器中也有按钮，显示剩余步数

---

## 第五部分：2D Tilemap 编辑模式（3 分钟）

### 步骤16：切换到 2D 模式

1. 在编辑器"编辑维度"区域，点击 **"切换到 2D 编辑"**
2. Scene 视图自动切换到 **2D 俯视角**（正交投影，正上方往下看）
3. 地图自动以 Unity Tilemap 渲染

### 步骤17：2D 模式下编辑

- 所有工具（笔刷/橡皮/矩形填充/洪水填充）与 3D 模式相同
- 地形精灵直接渲染到 Tilemap 上，所见即所得
- **切换回 3D**：点击"切换到 3D 编辑"

> 提示：2D 模式适合俯视角策略游戏（战棋、塔防、RPG 地图等）

### 步骤18：精灵尺寸匹配

如果你的精灵显示过大或过小（格子尺寸不匹配）：
1. 选中一个地形 → 编辑器会显示精灵参考信息
2. 如果显示"不匹配"警告 → 点击 **"自动匹配"** 按钮
3. 格子尺寸自动设为精灵的世界尺寸
4. 2D 模式下也会弹出全局检测对话框

---

## 第六部分：单位配置与放置（5 分钟）

### 步骤19：创建单位配置

每个 UnitConfig 资产 = 一种敌人/怪物/NPC。创建多个即可配置多种单位。

**方式一**：在编辑器中点击 **"+ 创建新单位配置"**，选择保存位置，自动创建

**方式二**：Project 窗口右键 → **Create → 地图编辑器 → 单位配置**，然后点击 **"扫描已存在的配置"**

**方式三**：从 Project 窗口拖入 UnitConfig 到编辑器的 ObjectField

### 步骤20：编辑单位属性

在 Project 窗口选中 UnitConfig，在 Inspector 中设置：
```
单位 ID:      goblin
单位名称:     哥布林
预制体:       （拖入怪物预制体）
基础攻击:     10
基础移速:     5
基础生命:     100
```

### 步骤21：创建对象层

1. 在编辑器"层级管理"区域，点击 **"+ 添加对象层"**
2. 层标签显示 `[对象层]`
3. 下方出现**单位选择区**

### 步骤22：在地图上放置单位

1. 在单位按钮行中点击要放置的单位（如"哥布林"）
2. 编辑模式选 **笔刷**（笔刷标签变为"笔刷（放置单位）"）
3. 在 Scene 视图格子上**左键点击** → 蓝色球体/圆盘标记出现
4. 切换回橡皮可移除单位

### 步骤23：地形层与对象层切换

- 使用 `◀ 上层` / `下层 ▶` 在地形层和对象层之间切换
- 地形层显示绿色/红色/黄色色块
- 对象层显示蓝色单位标记 + 单位名称标签

---

## 第七部分：保存与加载（2 分钟）

### 步骤24：保存地图

1. 点击 **"保存地图为 JSON"**
2. 选择保存路径 → 提示保存成功，显示地形格数和单位数量

### 步骤25：加载地图

1. 点击 **"从 JSON 加载地图"**
2. 选择 JSON 文件
3. 地形和单位数据一同恢复

### 步骤26：自动恢复

关闭编辑器窗口后重开 → 数据自动恢复（SessionState 持久化）

---

## 第八部分：运行时使用（5 分钟）

### 步骤27：让游戏加载地图

1. Hierarchy → Create Empty → 起名 `MapManager`
2. Add Component → `RuntimeMapGenerator`（或 `RuntimeTilemapGenerator` 用于大图）
3. 拖入 MapConfig 和 JSON 文件

### 步骤28：初始化全局配置

```csharp
using UnityEngine;
public class GameInitializer : MonoBehaviour
{
    public MapResourceConfig config;
    private void Awake()
    {
        GameMapGlobal.MapConfig = config;
    }
}
```

### 步骤29：代码中使用

```csharp
// 通行判断
bool canPass = GameMapGlobal.CheckUnitCanPass("grass", "knight");

// 读取地形属性
var t = GameMapGlobal.GetTerrain("grass");
int atk = t.GetInt("攻击加成");

// A* 寻路
var path = MapPathfinding.FindPath(0, 0, 10, 10, "knight", mapData);
```

## 常见问题排查

### Q：编辑器窗口空白？
**A**：拖入 MapConfig 到"地形资源配置表"字段。

### Q：Scene 视图看不到网格？
**A**：按 F 键聚焦，调整摄像机角度。

### Q：点击格子没反应？
**A**：确保选了地形/单位、编辑模式为笔刷、在 Scene 视图（非 Game 视图）中操作。

### Q：2D 模式精灵太大/太小？
**A**：点击"自动匹配"或调整精灵 PPU 导入设置。

### Q：怎么配置多种怪物？
**A**：创建多个 UnitConfig 资产（点击"+ 创建新单位配置"），每个代表一种敌人。

### Q：按 Ctrl+Z 没反应？
**A**：确保焦点在编辑器窗口或 Scene 视图上。

### Q：JSON 加载后地形不对？
**A**：检查配置表中的地形 ID 是否与 JSON 中一致。

### Q：大型地图卡顿？
**A**：使用 `RuntimeTilemapGenerator`（Tilemap 方案），性能远优于逐 GameObject。
