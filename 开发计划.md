# Unity 地形地图编辑器 - 开发计划

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 目标 | 开发一个 Unity 编辑器插件，实现 Scene 视图网格地图编辑、地形配置、JSON 序列化、运行时查询系统 |
| 类型 | Unity Editor 插件 + 运行时脚本库 |
| 最终交付 | 7 个 C# 脚本文件，可直接放入项目中运行 |
| 参考文档 | `地图编辑器设计文档.md` |
| Unity 版本 | 最低 2019.1，**推荐 2020.3 LTS 或更高** |

## 十、风险与应对

| 风险 | 概率 | 应对措施 |
|------|------|----------|
| SceneView.duringSceneGui 回调性能问题 | 低 | 网格线 LOD（远距离减少线条），使用 `EventType.Repaint` 过滤 |
| JSON 大文件性能（>10000 格） | 中 | 使用稀疏存储（只存非空格子）；后续可改 BinaryFormatter |
| `ScriptableObject` 在运行时修改后未持久化 | 低 | 编辑器使用 `EditorUtility.SetDirty()`，运行时不直接修改配置 |
| Undo 栈内存占用 | 中 | 增量存储方案（只存变更格子），限制最大 50 步 |
| 多层 + 寻路计算量 | 中 | 预构建 `Dictionary<(x,y,layer), cell>` 索引，O(1) 查询 |
| A* 寻路无解时的性能 | 低 | Open 集合边界限制 + 动态扩容避免预分配 |
| 编辑器状态丢失（关窗/脚本编译） | 中 | SessionState 自动保存/恢复 |
| 地形 ID 拼写错误静默失败 | 中 | 配置表 Validate 校验 + JSON 加载时 ID 有效性检查 |
| 大图每格一个 GameObject 内存爆炸 | 高 | 按地图规模选方案：<50×50 独立 GO / 50-200 Tilemap / >200 Chunk |
| 多地图场景全局配置冲突 | 低 | 提供 MapContext 实例化方案或 ID 前缀隔离方案 |
| Unity API 版本兼容性 | 低 | 文档明确标注最低版本 2019.1，推荐 2020.3 LTS |

## 十二、开发规范

### 命名规范
- 类名：PascalCase（如 `MapEditorWindow`）
- 方法名：PascalCase（如 `SaveMapToJson`）
- 私有字段：_camelCase（如 `_mapGrid`）
- 常量：UPPER_SNAKE（如 `MAX_HISTORY`）

### 代码组织
- 每个文件只包含一个主类 + 少量辅助类/枚举
- 编辑器代码放在 `Editor` 文件夹下，运行时代码放在 `Scripts` 下
- 使用 `#region` / `#endregion` 对长文件分段（可选）

### 提交规范
- 每个阶段完成后提交一次
- 提交信息格式：`[阶段X] 简要描述`
- 示例：`[阶段1] 完成编辑器窗口 + Scene 网格绘制`
