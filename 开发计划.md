# Unity 地形地图编辑器 - 开发计划

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 目标 | 开发一个 Unity 编辑器插件，实现 Scene 视图网格地图编辑、2D/3D 双模式、地形配置、单位放置、JSON 序列化、运行时查询系统 |
| 类型 | Unity Editor 插件 + 运行时脚本库 |
| 最终交付 | 12 个 C# 脚本文件，可直接放入项目中运行 |
| Unity 版本 | 最低 2019.1，**推荐 2020.3 LTS 或更高** |

---

## 二、开发阶段划分

```
阶段1 (核心编辑) → 阶段2 (运行时) → 阶段3 (扩展1-2) → 阶段4 (扩展3-6) → 阶段5 (质量测试) → 阶段6 (2D/单位)
      4天               1天                2天                  3天                  1天              3天
```

> **总计：14 个工作日**

## 十、风险与应对

| 风险 | 概率 | 应对措施 |
|------|------|----------|
| SceneView.duringSceneGui 回调性能问题 | 低 | 网格线 LOD，使用 EventType.Repaint 过滤 |
| JSON 大文件性能 | 中 | 稀疏存储（只存非空格子） |
| Undo 栈内存占用 | 中 | 增量存储，限制最大 50 步 |
| 精灵 PPU 与格子不匹配 | 高 | 自动检测 + 一键修正 + 全局对话框 |
| 2D/3D 模式切换数据丢失 | 低 | 数据模型共享 `_allLayers`，双向同步 |
| 对象层与地形层数据混淆 | 中 | 层类型标记 + 层切换时 UI 自动适配 |
| `System.Linq` 缺失导致编译错误 | 低 | 确保所有编辑器文件引用 `using System.Linq` |
