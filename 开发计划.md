# Unity 地形地图编辑器 - 开发计划

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 目标 | 开发一个 Unity 编辑器插件，实现 Scene 视图网格地图编辑、地形配置、JSON 序列化、运行时查询系统 |
| 类型 | Unity Editor 插件 + 运行时脚本库 |
| 最终交付 | 7 个 C# 脚本文件，可直接放入项目中运行 |
| 参考文档 | `地图编辑器设计文档.md` |

## 九、风险与应对

| 风险 | 概率 | 应对措施 |
|------|------|----------|
| SceneView.duringSceneGui 回调性能问题 | 低 | 控制网格线绘制量（大于 200×200 时降采样），使用 `EventType.Repaint` 过滤 |
| JSON 大文件性能（>10000 格） | 中 | 使用稀疏存储（只存非空格子）；后续可改 BinaryFormatter |
| `ScriptableObject` 在运行时修改后未持久化 | 低 | 编辑器使用 `EditorUtility.SetDirty()`，运行时不直接修改配置 |
| Undo 栈内存占用 | 低 | 限制最大 50 步，每步只存有数据的格子（稀疏快照） |
| 多层 + 寻路计算量 | 中 | 当前层查询逻辑 O(n)，大图可预构建 `Dictionary<(x,y,layer), cell>` 索引 |
| A* 寻路无解时的性能 | 低 | Open 集合有边界限制，最大遍历节点数 = 地图总面积 |

## 十、测试计划

### 单元测试（手动）

| 测试项 | 测试方法 |
|--------|----------|
| 地形 ID 查询 | 传入存在/不存在/空字符串的 terrainId，验证返回值 |
| 通行判断 | 分别测试 Passable / Impassable / UnitImpassable 三种类型 |
| 自定义属性读取 | 测试 Int/Float/Bool 取值 + 未注册属性名返回默认值 |
| JSON 序列化 | 保存后再加载，验证数据完全一致 |
| 多层查询 | 不同层同一坐标放不同地形，验证互不干扰 |

### 集成测试

| 测试项 | 测试方法 |
|--------|----------|
| 编辑器编辑 → 保存 → 加载 → 再编辑 | 完整操作流程 |
| 编辑器保存 JSON → 运行时加载生成地图 | 跨编辑器/运行时验证 |
| 单位走到不同地形 | 验证日志输出，属性加成正确 |
| A* 寻路 + 阻挡地形 | 验证路径绕开阻挡，终点不可达时返回空列表 |
| Undo → 继续编辑 → Undo | 验证 Undo 栈完整性，Redo 栈正确清空 |

### 边界测试

| 测试项 | 边界条件 |
|--------|----------|
| 空地图处理 | 0 个格子、空 JSON、空配置表 |
| 极端尺寸 | 1×1, 500×500, 1000×1000 |
| 配置为 null | 未初始化 GameMapGlobal.MapConfig 时的防御 |
| 最大 Undo 步数 | 超过 50 步后旧记录正确丢弃 |
| 拖拽出地图边界 | 越界格子不响应 |

## 十一、开发规范

### 命名规范
- 类名：PascalCase（如 `MapEditorWindow`）
- 方法名：PascalCase（如 `SaveMapToJson`）
- 私有字段：_camelCase（如 `_mapGrid`）
- 常量：UPPER_SNAKE（如 `MAX_HISTORY`）

### 代码组织
- 每个文件只包含一个主类 + 少量辅助类/枚举
- 编辑器代码放在 `Editor` 文件夹下，运行时代码放在 `Scripts` 下
- 使用 `#region` / `#endregion` 对长文件分段（可选）

### 提交规范
- 每个阶段完成后提交一次
- 提交信息格式：`[阶段X] 简要描述`
- 示例：`[阶段1] 完成编辑器窗口 + Scene 网格绘制`
