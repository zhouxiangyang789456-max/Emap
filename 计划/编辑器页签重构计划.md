# MapEditorWindow 页签重构 — 开发计划

## 问题

当前 `MapEditorWindow.OnGUI()` 将所有内容堆在一个长页面中，包含：
- 地图参数设置（名称/宽高/网格形状/格子尺寸/预设）
- 层级管理
- 2D/3D 切换
- 编辑模式选择
- 地形/单位选择
- 操作按钮（清空/保存/加载/撤销/重做）
- 地形资源配置列表（创建表单 + 折叠列表）
- 单位配置管理（创建/扫描/拖入/选择）

全部挤在一起，滚动很长，视觉混乱，操作效率低。

## 方案

拆分为 **3 个页签**，用 `EditorGUILayout.Toolbar` 实现顶部 Tab 切换：

```
┌─────────────────────────────────────────────────────────┐
│  [地图编辑]    [地形资源]    [单位资源]                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  （当前选中页签的内容）                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 页签 1：地图编辑

当前正在编辑的地图的所有设置和操作。

| 区域 | 内容 |
|------|------|
| 配置引用 | `_config` ObjectField（公共，始终显示在页签上方或独立行） |
| 地图基础参数 | 地图名称、宽度、高度 |
| 网格形状 | `GridShape` 下拉框 + 切换确认弹窗 |
| 格子尺寸 | cellSize 滑块 + 快速预设按钮（方形/六边形各自预设） |
| 六边形信息 | 仅六边形模式：格子总数、物理尺寸估算、坐标范围 |
| 地图统计 | 当前地图总格子数、当前层信息 |
| 层级管理 | 层切换滑块、添加/删除层按钮、层类型标签 |
| 编辑维度 | 2D/3D 切换 |
| 编辑模式 | 笔刷/橡皮/矩形(菱形)填充/洪水填充/范围填充 |
| 当前工具选择 | 地形选择下拉 / 单位选择按钮行 |
| 操作按钮 | 清空全部 / 保存JSON / 加载JSON |
| 撤销/重做 | 撤销(N) / 重做(N) 按钮 + Ctrl+Z/Y 快捷键 |

### 页签 2：地形资源

地形资源（TerrainResource）的创建、编辑、删除管理。

| 区域 | 内容 |
|------|------|
| 创建新地形 | "+ 创建新地形" 按钮 → 展开创建表单（prefab导入 + ID/名称/通行规则/精灵） |
| 校验 | "校验配置" 按钮 |
| 地形列表 | 折叠列表（foldout），每个地形可展开编辑详情（ID/名称/精灵/prefab/通行规则/自定义属性） |

### 页签 3：单位资源

单位配置（UnitConfig）的创建、扫描、管理。

| 区域 | 内容 |
|------|------|
| 创建/扫描 | "+ 创建新单位配置" + "扫描已存在的配置" |
| 拖入 | "拖入配置" ObjectField（接受 UnitConfig 资产拖入） |
| 单位列表 | 横向按钮排列，显示已加载的单位（名称+ID），点击选中 |
| 空状态 | 未加载任何单位时显示 HelpBox 提示 |

## 实现变更

### 改动范围

仅修改 **1 个文件**：`D:\git\Emap\Assets\Editor\MapEditor\MapEditorWindow.cs`

### 具体改动

#### 1. 增加 Tab 枚举和状态变量

```csharp
private enum EditorTab { MapEdit, TerrainResources, UnitResources }
private EditorTab _currentTab = EditorTab.MapEdit;
```

#### 2. 重构 OnGUI() 结构

```csharp
private void OnGUI()
{
    // 公共区域：配置引用（所有页签共用）
    _config = EditorGUILayout.ObjectField(...);
    if (_config == null) { HelpBox + return; }

    // 页签工具栏
    _currentTab = (EditorTab)GUILayout.Toolbar((int)_currentTab, new[] { "地图编辑", "地形资源", "单位资源" });

    // 分隔
    GUILayout.Space(8);

    switch (_currentTab)
    {
        case EditorTab.MapEdit:
            DrawTab_MapEdit();
            break;
        case EditorTab.TerrainResources:
            DrawTab_TerrainResources();
            break;
        case EditorTab.UnitResources:
            DrawTab_UnitResources();
            break;
    }

    HandleShortcuts(); // 快捷键始终有效
}
```

#### 3. 拆分现有方法到三个 Tab 方法

| 现有方法 | 移入 Tab |
|----------|---------|
| `DrawMapSetting()` (不含层管理的地形列表部分) | 地图编辑 |
| `DrawDimensionToggle()` | 地图编辑 |
| `DrawEditMode()` | 地图编辑 |
| `DrawTerrainSelect()` | 地图编辑 |
| `DrawUnitSelect()` | 地图编辑 |
| `DrawOperateBtn()` | 地图编辑 |
| 层管理部分（从 `DrawMapSetting` 中移出） | 地图编辑 |
| `DrawTerrainConfigList()` | 地形资源 |
| 单位配置管理（创建/扫描/拖入/列表） | 单位资源 |

#### 4. 需要从 DrawMapSetting() 中拆出的内容

`DrawMapSetting()` 目前包含：
- 地图基础参数 → 保留在地图编辑 Tab
- 网格形状 → 保留在地图编辑 Tab
- 格子尺寸 + 预设 → 保留在地图编辑 Tab
- 精灵尺寸参考 → 保留在地图编辑 Tab
- 地图统计 HelpBox → 保留在地图编辑 Tab
- 层级管理 → 保留在地图编辑 Tab

**无需拆分 DrawMapSetting()**，整个方法都放在地图编辑 Tab 中。

#### 5. 页签间联动

- 在地图编辑 Tab 中切换为对象层时，如果 `_unitConfigs` 为空 → 显示 HelpBox 提示去单位资源 Tab 加载
- 从地形资源 Tab 创建新地形后，切回地图编辑 Tab → 地形选择下拉自动更新（已有逻辑：`_selectTerrainIndex` 更新 + `Repaint()`）
- 快捷键 Ctrl+Z/Y 始终生效，不受 Tab 切换影响

## 不变的部分

以下逻辑完全不需要修改：
- Scene 视图渲染（`OnSceneGUI`、`DrawGridGizmos`、`DrawHoverPreview`、`HandleMouseInput`）
- 六边形/方形分支逻辑
- 撤销/重做系统
- JSON 导入导出
- SessionState 持久化
- `MapEditor2DSceneManager` 交互
- 所有 `internal` 访问器

## 验收标准

1. 顶部显示三个 Tab 按钮："地图编辑" / "地形资源" / "单位资源"
2. 默认打开"地图编辑" Tab
3. 点击 Tab 切换显示对应内容，无需滚动即可看到核心编辑控件
4. 在地形资源 Tab 创建新地形 → 切回地图编辑 → 下拉列表中可见新地形
5. 在单位资源 Tab 扫描单位 → 切回地图编辑 → 对象层可选单位
6. Ctrl+Z/Y 撤销重做在任何 Tab 下均生效
7. 配置引用（`_config` ObjectField）在所有 Tab 下均可见
8. `_config` 为空时，显示 HelpBox 提示，不显示 Tab
